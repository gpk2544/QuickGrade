Features
Smarter than keyword matching
QuickGrade uses RAG + LLM to understand what a student actually means — not just whether exact words appear.
📄

OCR Answer Extraction

Upload handwritten or printed answer sheets. The system extracts text automatically using document OCR — no manual typing needed.

🗄️

RAG — Textbook Knowledge Base

Upload your textbook PDF. It's chunked, embedded into vectors, and stored in ChromaDB. Every answer is graded against verified academic content.

🧠

LLM Concept-Based Grading

LLaMA 3 via Groq API evaluates each answer semantically. Paraphrased, synonym-heavy, or creatively written answers are scored fairly — not penalized.

✍️

Structured Feedback

Every student gets itemized feedback — covered concepts, missing points, and specific suggestions for improvement — not just a number.

📊

Partial Marking Engine

Marks are assigned proportionally based on concept coverage and explanation depth — not binary correct/incorrect like keyword systems.

📥

Excel Results Export

Download color-coded result sheets with per-question scores, totals, PASS/FAIL status, and class summary in one click.

How It Works
4 phases from paper to score
A modular, AI-driven pipeline combining RAG retrieval with LLM evaluation for intelligent, grounded grading.
1

UI & Authentication

Teacher creates a forum, uploads the question paper and textbook PDF. Firebase-secured login keeps data private.

2

Backend & Answer Processing

FastAPI receives student answers via API. OCR extracts text from uploaded answer sheets and sends it to the RAG module.

3

Knowledge Base & RAG Retrieval

Textbook is chunked and embedded into ChromaDB. Semantic search retrieves the most relevant reference content for each question.

4

LLM Evaluation & Auto-Grading

LLaMA 3 (via Groq API) receives the question, student answer, and retrieved reference. It assigns marks and generates structured feedback.

📄 Answer Sheet → 🔍 OCR → 🗄️ ChromaDB RAG → 🧠 Groq LLM → 📊 Score + Feedback
