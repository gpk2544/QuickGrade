================================================================================
                              DETAILED MODULE DESCRIPTIONS
================================================================================

1. BACKEND STRUCTURE
====================

main.py:
- FastAPI app initialization
- Firebase credentials loading
- CORS middleware configuration
- Route registration
- Exception handling

routes/:
- auth.py: Firebase token verification, user profile management
- forums.py: CRUD operations for forums, model answers
- evaluate.py: Student grading using LLM + RAG
- upload.py: File upload with OCR and RAG indexing
- export.py: Excel export functionality

services/:
- ocr_service.py: Tesseract OCR wrapper for text extraction
- pdf_parser.py: PyMuPDF-based PDF text extraction
- storage.py: Firebase Storage upload helper
- ai_grader.py: DEPRECATED - replaced by grading module
- rag_service.py: DEPRECATED - replaced by knowledge module

grading/ (Advanced LLM Grading):
- grader.py: Main facade orchestrating quality check → prompt → LLM → parse
- llm_client.py: Groq API client with model fallback
- prompt_builder.py: Structured prompt construction
- response_parser.py: JSON response parsing with repair
- quality_checker.py: Blank/short/garbled answer detection

knowledge/ (RAG Knowledge Base):
- pdf_loader.py: FitZ-based PDF text extraction
- chunker.py: Paragraph-aware overlapping chunking
- embedder.py: Sentence-transformers with cache
- vector_store.py: ChromaDB CRUD operations
- knowledge_base.py: Unified facade for KB operations

ocr/ (Advanced OCR - Optional):
- engines.py: EasyOCR, TrOCR, PaddleOCR implementations
- image_processor.py: Preprocessing (deskew, CLAHE, etc.)
- combiner.py: Multi-engine result merging
- pipeline.py: End-to-end OCR pipeline

rag/ (RAG Retrieval):
- retriever.py: Query expansion and semantic retrieval

storage/ (SQLite - Optional):
- database.py: Schema initialization and facade
- writer.py: Result storage operations
- reader.py: Result retrieval queries
- connection.py: WAL mode with retry logic

utils/:
- logger.py: Centralized logging
- retry.py: Retry decorator for API calls
- text.py: Text processing utilities

config/:
- settings.py: All configuration constants in dataclasses


2. FRONTEND STRUCTURE
=====================

index.html:
- Landing page with animated canvas
- Login/Register forms
- Dashboard with forum list
- Forum detail with student management
- Grading interface
- Results display

js/api.js:
- Centralized API client
- Auth token management
- All backend API calls
- Error handling

js/app.js:
- Single page application logic
- Page navigation
- Form handling
- Event listeners
